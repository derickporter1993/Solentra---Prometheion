## Screenshot Capture Instructions

### Prerequisites
- Access to a Salesforce org with Elaro installed
- Browser with developer tools for responsive design testing
- Screenshot tool (browser extension, OS screenshot, or design tool)

### Capture Guidelines

1. **Use Realistic Data**: Populate org with sample compliance data that represents real-world scenarios
2. **Anonymize Sensitive Information**: Remove or blur any PII, PHI, or sensitive data
3. **Consistent Styling**: Use consistent branding, colors, and UI elements
4. **High Resolution**: Capture at minimum 1920x1080 for desktop, 750x1334 for mobile
5. **Clean Interface**: Close unnecessary tabs, remove clutter, focus on key features
6. **Annotations**: Add callouts or labels to highlight important features (optional)

### Desktop Screenshots
- Use Chrome or Firefox browser
- Set browser zoom to 100%
- Capture full screen or specific component
- Include browser chrome (optional) or use clean view

### Mobile Screenshots
- Use browser developer tools device emulation
- Test on actual devices if possible (iPhone, Android)
- Capture at device-native resolution
- Show responsive design features

### Screenshot Tools
- **Browser Extensions**: Awesome Screenshot, Nimbus Screenshot
- **OS Tools**: macOS Screenshot (Cmd+Shift+4), Windows Snipping Tool
- **Design Tools**: Figma, Sketch (for annotated screenshots)

## File Naming Convention

- Use lowercase with hyphens: `dashboard-overview.png`
- Be descriptive: `compliance-scorecard.png` not `screenshot1.png`
- Match documentation references exactly
- Use PNG format for screenshots (JPG acceptable for photos)

## Image Optimization

Before adding to documentation:
- Compress images to reduce file size (aim for < 500KB per image)
- Use tools like TinyPNG, ImageOptim, or Squoosh
- Maintain quality while reducing file size
- Consider WebP format for better compression (with PNG fallback)
